# Contributing

Contributions are welcome!

Note that all issues and requests undergo a triage process, labels will be
applied to issues accordingly:

* `triage/needs-triage` - not accepted yet, needs maintainers to review
* `triage/needs-information` - more context, information or discussion needed
* `triage/accepted` - accepted and assigned to an upcoming milestone
* `triage/declined` - not accepted, the reasoning will be posted

## Bug reporting

To report bugs open an [issue].

[issue]:https://github.com/networking-incubator/coraza-kubernetes-operator/issues

## Small improvements

For small improvements, please feel free to submit a [pull request].

[pull request]:https://github.com/networking-incubator/coraza-kubernetes-operator/pulls

## Large improvements

For larger improvements and features, please post a [discussion] describing
what you're looking for so we can do some assessment together first.

[discussion]:https://github.com/networking-incubator/coraza-kubernetes-operator/discussions

## How to contribute patches

See: [DEVELOPMENT.md](DEVELOPMENT.md)

1. Fork the repository
2. Create a new branch (e.g. `git checkout -b feature/my-change`)
3. Make your changes
4. Commit with a clear message
5. Push to your fork
6. Open a pull request

> **Note**: For early feedback, or if your PR is incomplete, feel free to open
> it as a `draft` to indicate the need for early review and feedback.
